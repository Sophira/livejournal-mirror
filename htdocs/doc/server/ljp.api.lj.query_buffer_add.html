LJ::query_buffer_add

Name

LJ::query_buffer_add — Schedules an insert/update query to be run on a certain table sometime in the near future in a batch with a lot of similar updates, or immediately if the site doesn't provide query buffering. Returns nothing (no db error code) since there's the possibility it won't run immediately anyway.

Use

LJ::query_buffer_add(dbarg, table, query);

Arguments

  • dbarg. A master database handle ($dbh) or a master/slave set of database handles ($dbs) as generated by LJ::get_dbs.

  • table. Table to modify.

  • query. Query that'll update table. The query must not access any table other than that one, since the update is done inside an explicit table lock for performance.
