LJ::get_logtext

Name

LJ::get_logtext — Efficiently retrieves a large number of journal entry text, trying first slave database servers for recent items, then the master in cases of old items the slaves have already disposed of. See also: LJ::get_talktext.

Use

LJ::get_logtext(dbs, opts, itemid);

Arguments

  • dbs. A master/slave database pair, as generated by LJ::get_dbs.

  • opts. Optional hashref of special options. Currently only 'prefersubjects' key is supported, which returns subjects instead of events when there's a subject, and the subject always being undef.

  • itemid. List of itemids to retrieve the subject & text for.

Source:

cgi-bin/ljlib.pl

Returns:

hashref with keys being itemids, values being [ $subject, $body ]
