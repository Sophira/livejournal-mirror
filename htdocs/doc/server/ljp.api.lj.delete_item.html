LJ::delete_item

Name

LJ::delete_item — Deletes a journal item from a user's journal that resides in the old schema (cluster0).

Use

LJ::delete_item(dbarg, journalid, itemid, quick, deleter);

Arguments

  • dbarg. A master database handle ($dbh) or a master/slave set of database handles ($dbs) as generated by LJ::get_dbs.

  • journalid. Userid of journal to delete item from.

  • itemid. Itemid of item to delete.

  • quick. Optional flag to make the delete be a little quicker when many deletes are occuring. It just doesn't update lastitemid in userusage.

  • deleter. Optional code reference to run to handle a deletion. Mass-delete tools can use this to batch deletes in table locks for speed. Arguments to this coderef are ($tablename, $col, @ids). The default implementation is: "DELETE FROM $table WHERE $col IN (@ids)"

Info

This function is deprecated, just as the old schema is deprecated. In a few months this function will be removed. The new equivalent to this function is LJ::delete_item2.
