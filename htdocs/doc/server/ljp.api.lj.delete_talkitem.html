LJ::delete_talkitem

LJ::delete_talkitem — Deletes a comment (or multiple) and associated metadata.

Use

LJ::delete_talkitem(dbcm, journalid, jtalkid, light);

Arguments

  • dbcm. Cluster master db to delete item from.

  • journalid. Journalid (userid from user to delete comment from). The journal must reside on the $dbcm you provide.

  • jtalkid. The jtalkid of the comment. Or, an arrayref of jtalkids to delete multiple

  • light. boolean; if true, only mark entry as deleted, so children will thread.

Info

The tables talk2, talkprop2, talktext2, and dudata are all deleted from, immediately. Unlike LJ::delete_item2, there is no $quick flag to queue the delete for later, nor is one really necessary, since deleting from 4 tables won't be too slow.

Source:

cgi-bin/ljlib.pl

Returns:

boolean; number of items deleted on success (or zero-but-true), 0 on failure.
